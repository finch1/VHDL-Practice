Design Examples

The Nios II Embedded Design Suite (EDS) includes hardware and software design examples that work on Altera development boards. Each design example highlights the usage of one or more features. Source code is provided for you to examine and use as the starting point for your own embedded designs. Visit Nios II Embedded Processor Design Examples for additional and revised designs.

Hardware Design Examples

The hardware design examples illustrate how to combine a Nios II processor, memory, and peripherals into a practical embedded system. The design files for the Applications Processor MMU Design are provided under <Nios II EDS install path>/examples/verilog/<niosII_cycloneIII_3c120 or niosII_stratixIV_4sgx230>/applications_processor_mmu_design. The design files for the Triple Speed Ethernet Design are provided under <Nios II EDS install path>/examples/verilog/<niosII_cycloneIII_3c120 or niosII_stratixIV_4sgx230>/triple_speed_ethernet_design. The Ethernet Standard design is available on the Nios II Embedded Processor Design Examples page on the Altera website.

Triple Speed Ethernet Design

This design runs on an Altera development board. It showcases many of the advanced features of the Nios II processor, Triple Speed Ethernet and Scatter-Gather DMA.

Applications Processor MMU Design

This design runs on an Altera development board. It showcases many of the advanced features of the Nios II processor with the MMU feature, Triple Speed Ethernet and Scatter-Gather DMA. Full-featured operating systems such as Linux can run on this design.

Ethernet Standard Design

This design is provided to showcase the Nios II Ethernet Standard design running on Altera development boards. This simple, straightforward design highlights many of the features of the Nios II processor, Triple Speed Ethernet and Scatter-Gather DMA.

Software Design Examples

The software design examples demonstrate how to write programs that take advantage of various Nios II processor features. Each example provides its design files and readme.txt documentation file in a directory under <Nios II EDS install path>/examples/software. The design examples are provided as project templates in the Nios II Software Build Tools for Eclipse, and as command-line examples for the Nios II Software Build Tools.

To create a software example in the Nios II Software Build Tools for Eclipse, use the Nios II Application and BSP from Template command, and choose the desired project template.

A list of the software design examples is provided below. For more information about each example design, refer to the corresponding readme.txt file.

  • Hello World - The "Hello World" program based on a hosted C environment.
  • Blank Project - A C/C++ project with no C files.
  • Board Diagnostic - Exercises most of the peripherals on the Altera development board.
  • Count Binary - Displays a count pattern on the Altera development board LEDs, 7-segment display, and LCD display. Detects button presses using an interrupt service routine.
  • Hello Alt_main - Demonstrates using a free-standing (i.e. non-hosted) C environment with manual driver initialization.
  • Hello MicroC/OS-II - "Hello World" based on the MicroC/OS-II RTOS.
  • Hello World Small - "Hello World" program with the smallest possible memory footprint. Demonstrates using standard compiler options to reduce memory footprint.
  • Memory Test - Writes and reads RAM and flash memories.
  • Memory Test Small - Memory Test program with the smallest possible memory footprint.
  • Simple Socket Server - A telnet application based on the sockets interface provided by the NicheStack TCP/IP Stack - Nios II Edition running on MicroC/OS-II. This example accompanies the Using the NicheStack TCP/IP Stack Nios II Edition Tutorial. 
  • Web Server - A web server based on the sockets API provided by the NicheStack TCP/IP Stack - Nios II Edition running on MicroC/OS-II. This example demonstrates remote configuration over Ethernet and control of various board components over Ethernet.
